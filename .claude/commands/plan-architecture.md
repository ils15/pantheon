# Plan Architecture (Athena)

Plan system architecture with strategic research and a TDD-driven implementation plan.

**Topic:** $ARGUMENTS

## Planning Process

### 1. Understand Requirements
- What to build and how it connects to the existing system
- Constraints (tech, time, security)

### 2. Research (Parallel Discovery)
- Discover related files and patterns
- Understand dependencies
- Identify existing conventions

### 3. Create TDD Plan
Must include:
- **Overview:** what will be built, success criteria
- **3–5 Phases:** each self-contained with tests
- **Per Phase:** tests to write (RED) → minimal code (GREEN) → files to modify
- **Risks:** what could go wrong and mitigation

### 4. Output Format
- Requirements summary
- Codebase findings
- Comprehensive TDD plan
- Risk assessment
- Design decisions with rationale
- Ready for Zeus handoff

After plan, offer: "Ready to implement? Pass this plan to Zeus."
