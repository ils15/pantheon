# Sketch — Idea to Spec

Turn a rough idea into a structured feature spec through a short Q&A interview.

**Idea:** $ARGUMENTS

## Interview Protocol

Ask 3–5 targeted questions **one at a time** in this order:

1. **Scope** — "What's the minimum viable version of this feature?"
2. **Users** — "Who uses this and what's their primary workflow?"
3. **Constraints** — "Are there tech, time, or integration constraints?"
4. **Success criteria** — "How will you know this is working correctly?"
5. **Non-goals** — "What should this explicitly NOT do?"

Rules:
- Ask **one question at a time** — never batch
- Stop after at least 3 answers
- Confirm understanding before writing the spec

## Spec Output Format

```markdown
# Feature Spec: <Name>
**Date:** YYYY-MM-DD  **Status:** Draft

## Overview
## Goals
## Non-Goals
## Functional Requirements (Must Have / Should Have / Won't Have)
## Technical Constraints
## Success Criteria
## Open Questions
## Suggested Implementation Phases
```
